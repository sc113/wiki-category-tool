# Wiki Category Tool

Wiki Category Tool — десктоп-приложение на `PySide6` для пакетной работы со страницами и категориями Wikimedia через `pywikibot` и MediaWiki API.

Проект ориентирован на сценарии массовой обработки TSV:

- считывание списков страниц и подкатегорий;
- массовая перезапись существующих страниц;
- массовое создание отсутствующих страниц;
- переименование страниц с переносом категорий;
- удаление избыточных категорий;
- межъязыковая синхронизация категорий.

---

## Основные возможности

- Авторизация в проектах Wikimedia через `pywikibot` (с сохранением конфигурации в `configs/`).
- Работа с пространствами имён и префиксами (`Авто` или принудительная нормализация к выбранному NS).
- Предпросмотр операций перед запуском записи.
- Фоновые воркеры для сетевых и массовых операций (GUI не блокируется).
- Логи выполнения и debug-диалог.
- Кэш namespace-префиксов и правил шаблонных замен.

## Форматы TSV

### Перезапись / Создание

```text
Title\tLine 1\tLine 2\t...
```

- Первая колонка: заголовок страницы.
- Остальные колонки объединяются в конечный текст через переносы строк.
- Рекомендуемая кодировка: `UTF-8 with BOM`.

### Переименование

```text
OldTitle\tNewTitle\tComment
```

- `Comment` (3-я колонка) опционален.
- Общий комментарий во вкладке может переопределить комментарий из TSV.

### Удаление избыточных категорий

```text
PreciseCategory\tBroadCategory
```

## Вкладки интерфейса

- `Авторизация` — вход, проект/язык, принудительное обновление namespace-префиксов.
- `Считывание` — сбор страниц/подкатегорий, экспорт списка.
- `Перезапись` — массовое обновление существующих страниц из TSV.
- `Создание` — создание только отсутствующих страниц из TSV.
- `Переименование` — rename + перенос категорий (включая шаблонные параметры и review-диалог).
- `Избыточные категории` — удаление broad-категорий при наличии precise.
- `Синхронизация категорий` — перенос категорий между языками.
- `Обзор` — состояние сессии, UI-настройки, статистика.

## Конфиги и runtime-данные

Папка `configs/` создаётся автоматически и хранит:

- `user-config.py`, `user-password.py` — настройки `pywikibot`;
- `pywikibot*.lwp` — cookies;
- `template_rules.json` — правила шаблонных замен;
- `update_settings.json` — настройки проверки обновлений;
- `apicache/` — кэш API и namespace-префиксов.

Важно: это окружение-зависимые данные; обычно их не коммитят в репозиторий.

---

## Структура проекта

- `gui/` — главное окно, вкладки, диалоги, виджеты.
- `workers/` — фоновые потоки операций.
- `core/` — API-клиент, namespace manager, локализация, конфигурация.
- `locales/` — словари интерфейса (`ru-RU.json`, `en-US.json`).
- `configs/` — runtime-состояние (создаётся при запуске).

## Практические замечания

- Перед массовыми изменениями тестируйте на небольшом наборе страниц.
- Учитывайте локальные правила конкретного Wikimedia-проекта.
- Для write-операций важны права учётной записи и корректный edit summary.
